# TinySkies — Requirements

Source: `tinyskies source project` (a.k.a. "GlobeFly" — repo's internal name; deployed product is **TinySkies** at https://tinyskies.vercel.app). User confirmed `tinyskies` as the canonical project name on 2026-05-05.

Scope locked with the user on 2026-05-05:

- **MVP slice (1A):** core flight on a globe + paintball, single-player only.
- **Multiplayer (2A):** deferred. No remote players, no Socket.io, no Colyseus glue in this pass.
- **Persistence (3):** skipped. No progression, no upgrades, no leaderboard.
- **Assets (4):** reuse original source code/values verbatim — biplane is procedural; globe is procedural; no GLBs needed for MVP.
- **Fidelity (5):** behave identically — flight feel, camera feel, and paintball feel must match the source. All tuning constants pulled directly from `client/src/game/Plane.ts`, `client/src/game/CameraRig.ts`, `client/src/game/SphericalMath.ts`, `shared/types.ts`.

Requirement IDs are stable. `MVP` priority = required for first playable. `Later` priority = explicitly deferred from this pass.
